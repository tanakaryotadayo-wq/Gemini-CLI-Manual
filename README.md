# Gemini CLI スラッシュコマンド 完全ガイド

Gemini CLIでは、対話プロンプトに `/` から始まるコマンドを入力することで、セッションの制御やツールの呼び出し、設定の変更などが行えます。

このドキュメントでは、標準のGemini CLIコマンドに加え、現在の環境（Antigravity IDE）で特別に設定されているカスタムコマンドについても解説します。

---

## 🚀 1. Antigravity IDE カスタムコマンド
（あなたの環境に特化した、強力なオーケストレーション・コマンドです）

### `/jules <task>`
- **説明**: 独立したタスクをGitHub Issue経由で非同期ワーカー（Julesエージェント）に投下し、バックグラウンドで処理させます。
- **現実的な使い方**: 「`/jules ログイン画面のバリデーションエラーを修正して`」のように入力し、自分は別の作業を進める。
- **応用的な使い方**: 大きな機能追加を複数の小さなIssueに分割し、複数の `/jules` コマンドを連続して叩くことで、複数エージェントに並列で実装を進めさせる。

### `/deepthink <query>`
- **説明**: 困難なアーキテクチャ設計やPRの論理バグ監査など、深く複雑な思考が必要なタスクをGemini CLIのDEEPTHINKモードで実行します。
- **現実的な使い方**: 「`/deepthink このリポジトリのDB接続プールの枯渇問題の根本原因を特定して、最適な解決策を提案して`」
- **応用的な使い方**: 実装前の設計フェーズで呼び出し、エッジケースやセキュリティ上の懸念点、パフォーマンスのボトルネックを徹底的に洗い出させる。

### `/deepsearch <query>`
- **説明**: ドキュメントのないフレームワークや最新技術情報、エラーの解決策などをGemini CLI経由で徹底調査します。
- **現実的な使い方**: 「`/deepsearch 最近のNext.js App Routerでのキャッシュパージのベストプラクティスを調べて`」
- **応用的な使い方**: 未知のライブラリや難解なエラーログを投げ込み、一次ソース（公式ドキュメントやGitHubのIssue）に基づく信頼性の高い解決手順をレポートさせる。

### `/orchestrate`
- **説明**: 現在のプロジェクトの未解決Issue群をDEEPTHINKで分析し、最も適切なものを自動抽出して、Julesの複数スレッドに同時に投下する全自動パイプラインを起動します。
- **現実的な使い方**: プロジェクトのタスクが溜まってきた時に ` /orchestrate` を実行し、AI群に自律的にタスクを消化させる。
- **応用的な使い方**: 朝の業務開始時に実行し、自分がコードレビューをするためのPRを日中にAIに自動生成させておく。

### `/nightMode` (コマンドパレットから実行)
- **説明**: 寝る前に実行するためのコマンド。一晩かけてJulesに複数のIssueを解決させ、翌朝までにレビュー待ちのPRを生成させます。
- **現実的な使い方**: 1日の作業終わりに実行し、残っている細かいバグ修正やリファクタリングを夜間に終わらせる。

## 🛠️ 2. コア・基本コマンド（Core Commands）

### `/help` または `/?`
- **説明**: 利用可能なすべてのコマンドと簡単な使い方を画面に表示します。
- **使い方**: 迷ったらまずこれを入力します。

### `/clear`
- **説明**: ターミナルの画面とスクロール履歴をクリアします（ショートカット: `Ctrl+L`）。
- **使い方**: 画面の文字が多くなりすぎて見づらくなった時に使います。

### `/quit` または `/exit`
- **説明**: Gemini CLIのセッションを終了して、通常のターミナルに戻ります。

### `/about`
- **説明**: バージョン情報やシステム情報を表示します。
- **応用的な使い方**: バグ報告や環境のトラブルシューティング時に、自分の環境情報を確認するために使います。

### `/docs`
- **説明**: ブラウザで公式ドキュメントを開きます。

## ⏳ 3. セッションと履歴（Session & History）

### `/chat` または `/resume`
- **説明**: 会話セッション（チャットの履歴やコンテキスト）を管理します。
- **サブコマンド**:
  - `/chat list`: 保存されているチェックポイント（履歴）の一覧を表示します。
  - `/chat save <タグ名>`: 現在の状態に名前をつけて保存します。（例: `/chat save feature-auth-done`）
  - `/chat resume <タグ名>`: 保存したチェックポイントから再開します。
  - `/chat share`: チャット履歴をMarkdownやJSONとして出力（エクスポート）します。
- **応用的な使い方**: 大きな変更を加える前に `/chat save before-refactor` で保存し、失敗したら `/chat resume before-refactor` で即座にやり直す。

### `/rewind`
- **説明**: チャットの履歴を1つ前に戻し、AIの返答やコードの変更をなかったことにします（ショートカット: `Esc` を2回押す）。
- **使い方**: AIが的外れな回答をした直後に使って、別の指示を出し直します。

### `/restore`
- **説明**: ツールの実行（ファイル編集など）によって変更されたファイルを、以前の状態に戻します。
- **使い方**: AIがファイルを壊してしまった場合に、直前の安全な状態に復元します。

### `/compress`
- **説明**: これまでの会話のコンテキスト（文脈）を要約し、トークン消費量を節約します。
- **応用的な使い方**: 会話が長引き、動作が重くなったりAPI料金が気になり始めたタイミングで実行し、必要な情報だけを残して履歴を軽くします。

### `/copy`
- **説明**: 直前のAIの回答をクリップボードにコピーします。

## ⚙️ 4. 設定とワークスペース（Configuration & Workspace）

### `/settings`
- **説明**: 対話型の設定エディタを開きます。

### `/model`
- **説明**: 使用するAIモデルを変更します。
- **現実的な使い方**: `/model set gemini-2.5-pro` のように入力して、より賢いモデルや高速なモデルに切り替えます。
- **注意点**: Antigravity IDEのルールではメインはClaudeとされており、Geminiのモデル選びには制限がある場合があります（Gemini 3.1 Pro/Flash推奨）。

### `/directory` または `/dir`
- **説明**: AIに読み込ませるワークスペース（フォルダ）を追加・確認します。
- **応用的な使い方**: 複数のプロジェクトを跨いで作業する場合に、`/dir add /path/to/another-repo` として、両方のコードをAIに理解させながら作業します。

### `/init`
- **説明**: 現在のディレクトリに `GEMINI.md` というファイルを生成します。このファイルにはプロジェクト固有のルールやコンテキストを書き込みます。
- **使い方**: 新しいプロジェクトを始めたら最初に実行し、「このプロジェクトはTypeScriptとReactを使っている」「変数の命名規則はこれ」といった指示を書いておきます。

### `/theme` / `/vim`
- **説明**: CLIの見た目（テーマ）を変更したり、Vim風のキーバインド（操作方法）をオン/オフします。

## 🔌 5. ツールと拡張機能（Tools & Extensions）

### `/tools`
- **説明**: AIが使用できるツール（ファイル読み書き、検索、コマンド実行など）の一覧を表示します。`/tools desc` で技術的な詳細が確認できます。

### `/skills`
- **説明**: エージェントの専門スキルを管理します（有効化、無効化、一覧表示）。
- **使い方**: `/skills list` で使えるスキルを確認し、特定の技術（Azureや特定のフレームワーク等）に関するタスクを投げる前に確認します。

### `/mcp` / `/extensions` / `/agents`
- **説明**: MCP（Model Context Protocol）サーバー、CLI拡張機能、サブエージェントを管理するためのコマンドです。
- **応用的な使い方**: 外部のAPIやデータベース、独自の社内ツールなどをGemini CLIに連携させる高度な設定を行う際に使用します。

### `/commands`
- **説明**: ユーザー定義のカスタムコマンドを管理します。`.toml`ファイルで定義したコマンドを読み込み直す場合は `/commands reload` を使います。

## 💡 7. 便利な構文（コマンド以外）

対話プロンプトで以下の構文も使えます：

### `@<ファイルパス>`
- **説明**: 指定したファイルやフォルダの中身を、プロンプトに直接挿入（読み込み）します。
- **現実的な使い方**: 「`@src/main.ts このコードのリファクタリングをして`」のように、見せたいファイルを明示的に 指定して指示を出します。

### `!<コマンド>`
- **説明**: CLIを終了せずに、通常のシェルコマンド（ターミナルコマンド）を実行します。
- **現実的な使い方**: 「`!git status`」や「`!ls -la`」など、ちょっとした確認を行いたい時に非常に便利です。

## 😈 8. 応用・裏コマンド（Advanced & Undocumented Features）
公式ドキュメントにはあまり書かれていない、玄人向けの高度な使い方です。

### `/mcp` (Model Context Protocol) の高度な運用
外部ツールやサーバーをAIに繋ぐMCP機能の裏コマンドです。
- **`/mcp auth [サーバー名]`**: リモートサーバー（IAPで保護されたCloud Runなど）のOAuth認証フローをトリガーします。
- **`/mcp schema`**: 接続中のすべてのMCPツールの完全なJSONスキーマをダンプして確認します（デバッグ用）。
- **`/mcp enable/disable <名前> --session`**: グローバル設定を汚さずに、**現在のセッション（起動中）だけ**特定のMCPサーバーをオン/オフします。
- **`@server://resource/path`**: プロンプト内で直接MCPが公開しているリソース（APIドキュメントやリアルタイムログなど）を読み込ませるための特殊構文です。

### `/agents` (サブエージェント) の強制ルーティング
AIの「専門家（サブエージェント）」を使いこなす機能です。
- **`@<エージェント名>` (例: `@codebase_investigator`)**: プロンプトの先頭につけることで、メインAIの自動判断をバイパスし、**強制的に指定した専門エージェントにタスクを処理**させます。
- **`/agents config <名前>`**: 特定のエージェントのモデル、Temperature（創造性）、ターン制限などを対話的に即座に変更できます。

### `/skills` (専門スキル) のローカル開発
- **`/skills link <パス>`**: ローカルにあるスキルフォルダのシンボリックリンクを作成し、再インストールなしでスキルの開発・テストができるようにします。

### `/commands` (カスタムコマンド) の変態的記法
`.toml` で自作コマンドを作る際に使える強力な構文です。
- **名前空間**: コマンドのファイルをフォルダに分けると（例: `.gemini/commands/git/commit.toml`）、CLI上では `/git:commit` のように階層化して呼び出せます。
- **`{{args}}`**: ユーザーの入力をコマンドの引数として注入します。シェル実行ブロック `!{}` の中で使うと、OSコマンドインジェクションを防ぐために**自動でシェルエスケープ**されます。
- **マルチモーダル注入**: `@{画像パス}` や `@{PDFパス}` をコマンド内に仕込むと、テキストだけでなく画像や音声、PDFなどのバイナリデータも直接プロンプトに叩き込めます。

### `/extensions` (拡張機能) の開発と設定
- **`/extensions link <パス>`**: 開発中のローカル拡張機能をシンボリックリンクで読み込みます（ホットリロード開発用）。
- **`/extensions config <名前>`**: 拡張機能が必要とする環境変数（APIキーなど）を、システムのキーチェーン（MacならKeychain Access）に安全に保存・設定するためのダイアログを開きます。

### `/chat` (セッション管理) のデバッグ機能
- **`/chat debug`**: AIとサーバーの間でやり取りされた「直前のAPIリクエストの生データ（JSONペイロード）」をダンプします。プロンプトがどう解釈されたかを確認したい時に必須です。

### `/plan` (設計モード) の応用
- **完全な安全モード**: `/plan` を実行中は、ユーザーが明示的に「承認（Approve）」するまで、**AIはファイルシステムを一切変更（破壊）できない**リードオンリー状態にロックされます。
- **`/plan copy`**: AIが作成してユーザーが承認した「作業計画書」をクリップボードにコピーします。これを別のLLMに渡したり、GitHub Issueの仕様書として貼り付けるのに便利です。

## 🧠 9. アーキテクチャと結合ロジック (Architecture & Integration Logic)
各機能がシステム内部でどのようにパース（解析）され、実行されているかの技術的定義です。これらを理解することで、複数の機能を組み合わせた「自分だけの最強の自動化パイプライン」を構築できます。

### 1. カスタムコマンド（Custom Commands）のパースと注入
- **設定ファイル**: `~/.gemini/commands/` またはプロジェクト内の `.gemini/commands/` に配置された `.toml` ファイル。
- **スキーマ定義**:
  ```toml
  description = "コマンドの説明"
  prompt = "メインのプロンプトテキスト {{args}}"
  ```
- **実行ロジック**:
  1. CLIがTOMLをパースし、プレースホルダーを特定します。
  2. **Injection（注入）**: `{{args}}` はユーザー入力に置換されます。`@{パス}` はファイルの中身に、`!{コマンド}` はシェルコマンドの実行結果（標準出力）に置換されます。**これはLLMにプロンプトが送信される「前」にCLI側で処理されます。**
  3. 全てが文字列として解決された後、ユーザーメッセージとしてAIに送信されます。

### 2. サブエージェント（Subagents）の独立コンテキスト
サブエージェントは、メインAIとは別の「独立した思考ループ」を持つ専門家です。
- **スキーマ定義**: Markdownファイルの上部にYAMLフロントマター（Frontmatter）で定義します。
  ```markdown
  ---
  name: security-auditor
  description: "セキュリティの専門家"
  tools: ["read_file", "mcp_*"]
  model: "gemini-3-flash-preview"
  max_turns: 10
  ---
  ここから下にシステムプロンプト（指示）を書く...
  ```
- **実行ロジック**:
  サブエージェントは、メインAIからは**「1つのツール」として認識**されます。メインAIがこのツールを呼び出すと、CLIは新しい `LocalAgentExecutor` を裏で起動します。この新しいエグゼキューターは、指定されたシステムプロンプトと許可されたツールセット（`tools`）だけを持ち、メインの会話履歴（コンテキスト）を汚染することなくタスクを完遂し、結果だけをメインAIに返します。

### 3. MCP (Model Context Protocol) の動的ディスカバリ
外部ツールをAIに接続するためのプロトコルです。
- **実行ロジック**:
  1. **Discovery Layer**: CLIは起動時（または `/mcp reload` 時）に指定されたサーバーと Stdio, SSE, HTTP などで通信し、利用可能なツールのJSONスキーマを取得します。
  2. **Namespace**: 取得したツールは `mcp_{サーバー名}_{ツール名}` という名前空間でグローバルレジストリに登録されます。
  3. **Execution Layer**: AIがツールを呼び出すと、CLIはリクエストをインターセプトし、ユーザーの承認（設定で `trust: true` にしていない場合）を求めた後、サーバーにリクエストを転送します。

### 4. フック（Hooks）による同期インターセプト
AIの思考ループ（エージェントループ）の途中に割り込んで、処理を変更・ブロックする仕組みです。
- **実行ロジック**:
  1. `BeforeTool`（ツール実行前）や `AfterModel`（モデル応答後）などのライフサイクルイベントで、外部スクリプト（Pythonやシェルスクリプト）がトリガーされます。
  2. CLIはスクリプトの `stdin`（標準入力）にJSONペイロードを流し込みます。
  3. スクリプトは必ず `stdout`（標準出力）にJSONを返さなければなりません。
  4. **終了コード**: スクリプトが `0` で終了すれば続行、`2` で終了すれば「System Block（強制停止）」となり、AIの行動を強制的にキャンセルさせることができます。

### 5. 拡張機能（Extensions）のバンドル
上記のすべて（コマンド、サブエージェント、MCPサーバー、フック）を1つのパッケージにまとめたものです。
- **スキーマ (`gemini-extension.json`)**:
  ```json
  {
    "name": "my-extension",
    "version": "1.0.0",
    "mcpServers": { "my-db": { "command": "python3", "args": ["db_server.py"] } },
    "excludeTools": ["write_file"],
    "settings": [{ "name": "API_KEY", "envVar": "MY_API_KEY", "sensitive": true }]
  }
  ```
- **実行ロジック**: 拡張機能はロード時にグローバル設定とマージされます。名前の衝突が起きた場合は拡張機能側が「拡張機能名」で名前空間化されます。

### 🔥 高度な自動化の構築例（組み合わせ技）
これらの仕様を理解すると、以下のような変態的なパイプラインを構築できます。
1. 特定のデータベースに繋がる **MCPサーバー** を同梱した **拡張機能** を作る。
2. その拡張機能内に、そのMCPツールだけを使える（`tools: ["mcp_my-db_*"]`）特化型の **サブエージェント** を定義する。
3. **フック（Hooks）** を使って、プロジェクトの状態に応じて動的にプロンプトを注入する。
4. **カスタムコマンド（.toml）** で `!{}` を使って現在のGitの差分を取得し、`{{args}}` と一緒に **サブエージェント** (`@サブエージェント名`) に丸投げするワンライナーコマンドを作る。
