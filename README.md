# Gemini CLI スラッシュコマンド 完全ガイド

Gemini CLIでは、対話プロンプトに `/` から始まるコマンドを入力することで、セッションの制御やツールの呼び出し、設定の変更などが行えます。

このドキュメントでは、標準のGemini CLIコマンドに加え、現在の環境（Antigravity IDE）で特別に設定されているカスタムコマンドについても解説します。

---

## 🚀 1. Antigravity IDE カスタムコマンド
（あなたの環境に特化した、強力なオーケストレーション・コマンドです）

### `/jules <task>`
- **説明**: 独立したタスクをGitHub Issue経由で非同期ワーカー（Julesエージェント）に投下し、バックグラウンドで処理させます。
- **現実的な使い方**: 「`/jules ログイン画面のバリデーションエラーを修正して`」のように入力し、自分は別の作業を進める。
- **応用的な使い方**: 大きな機能追加を複数の小さなIssueに分割し、複数の `/jules` コマンドを連続して叩くことで、複数エージェントに並列で実装を進めさせる。

### `/deepthink <query>`
- **説明**: 困難なアーキテクチャ設計やPRの論理バグ監査など、深く複雑な思考が必要なタスクをGemini CLIのDEEPTHINKモードで実行します。
- **現実的な使い方**: 「`/deepthink このリポジトリのDB接続プールの枯渇問題の根本原因を特定して、最適な解決策を提案して`」
- **応用的な使い方**: 実装前の設計フェーズで呼び出し、エッジケースやセキュリティ上の懸念点、パフォーマンスのボトルネックを徹底的に洗い出させる。

### `/deepsearch <query>`
- **説明**: ドキュメントのないフレームワークや最新技術情報、エラーの解決策などをGemini CLI経由で徹底調査します。
- **現実的な使い方**: 「`/deepsearch 最近のNext.js App Routerでのキャッシュパージのベストプラクティスを調べて`」
- **応用的な使い方**: 未知のライブラリや難解なエラーログを投げ込み、一次ソース（公式ドキュメントやGitHubのIssue）に基づく信頼性の高い解決手順をレポートさせる。

### `/orchestrate`
- **説明**: 現在のプロジェクトの未解決Issue群をDEEPTHINKで分析し、最も適切なものを自動抽出して、Julesの複数スレッドに同時に投下する全自動パイプラインを起動します。
- **現実的な使い方**: プロジェクトのタスクが溜まってきた時に ` /orchestrate` を実行し、AI群に自律的にタスクを消化させる。
- **応用的な使い方**: 朝の業務開始時に実行し、自分がコードレビューをするためのPRを日中にAIに自動生成させておく。

### `/nightMode` (コマンドパレットから実行)
- **説明**: 寝る前に実行するためのコマンド。一晩かけてJulesに複数のIssueを解決させ、翌朝までにレビュー待ちのPRを生成させます。
- **現実的な使い方**: 1日の作業終わりに実行し、残っている細かいバグ修正やリファクタリングを夜間に終わらせる。

## 🛠️ 2. コア・基本コマンド（Core Commands）

### `/help` または `/?`
- **説明**: 利用可能なすべてのコマンドと簡単な使い方を画面に表示します。
- **使い方**: 迷ったらまずこれを入力します。

### `/clear`
- **説明**: ターミナルの画面とスクロール履歴をクリアします（ショートカット: `Ctrl+L`）。
- **使い方**: 画面の文字が多くなりすぎて見づらくなった時に使います。

### `/quit` または `/exit`
- **説明**: Gemini CLIのセッションを終了して、通常のターミナルに戻ります。

### `/about`
- **説明**: バージョン情報やシステム情報を表示します。
- **応用的な使い方**: バグ報告や環境のトラブルシューティング時に、自分の環境情報を確認するために使います。

### `/docs`
- **説明**: ブラウザで公式ドキュメントを開きます。

## ⏳ 3. セッションと履歴（Session & History）

### `/chat` または `/resume`
- **説明**: 会話セッション（チャットの履歴やコンテキスト）を管理します。
- **サブコマンド**:
  - `/chat list`: 保存されているチェックポイント（履歴）の一覧を表示します。
  - `/chat save <タグ名>`: 現在の状態に名前をつけて保存します。（例: `/chat save feature-auth-done`）
  - `/chat resume <タグ名>`: 保存したチェックポイントから再開します。
  - `/chat share`: チャット履歴をMarkdownやJSONとして出力（エクスポート）します。
- **応用的な使い方**: 大きな変更を加える前に `/chat save before-refactor` で保存し、失敗したら `/chat resume before-refactor` で即座にやり直す。

### `/rewind`
- **説明**: チャットの履歴を1つ前に戻し、AIの返答やコードの変更をなかったことにします（ショートカット: `Esc` を2回押す）。
- **使い方**: AIが的外れな回答をした直後に使って、別の指示を出し直します。

### `/restore`
- **説明**: ツールの実行（ファイル編集など）によって変更されたファイルを、以前の状態に戻します。
- **使い方**: AIがファイルを壊してしまった場合に、直前の安全な状態に復元します。

### `/compress`
- **説明**: これまでの会話のコンテキスト（文脈）を要約し、トークン消費量を節約します。
- **応用的な使い方**: 会話が長引き、動作が重くなったりAPI料金が気になり始めたタイミングで実行し、必要な情報だけを残して履歴を軽くします。

### `/copy`
- **説明**: 直前のAIの回答をクリップボードにコピーします。

## ⚙️ 4. 設定とワークスペース（Configuration & Workspace）

### `/settings`
- **説明**: 対話型の設定エディタを開きます。

### `/model`
- **説明**: 使用するAIモデルを変更します。
- **現実的な使い方**: `/model set gemini-2.5-pro` のように入力して、より賢いモデルや高速なモデルに切り替えます。
- **注意点**: Antigravity IDEのルールではメインはClaudeとされており、Geminiのモデル選びには制限がある場合があります（Gemini 3.1 Pro/Flash推奨）。

### `/directory` または `/dir`
- **説明**: AIに読み込ませるワークスペース（フォルダ）を追加・確認します。
- **応用的な使い方**: 複数のプロジェクトを跨いで作業する場合に、`/dir add /path/to/another-repo` として、両方のコードをAIに理解させながら作業します。

### `/init`
- **説明**: 現在のディレクトリに `GEMINI.md` というファイルを生成します。このファイルにはプロジェクト固有のルールやコンテキストを書き込みます。
- **使い方**: 新しいプロジェクトを始めたら最初に実行し、「このプロジェクトはTypeScriptとReactを使っている」「変数の命名規則はこれ」といった指示を書いておきます。

### `/theme` / `/vim`
- **説明**: CLIの見た目（テーマ）を変更したり、Vim風のキーバインド（操作方法）をオン/オフします。

## 🔌 5. ツールと拡張機能（Tools & Extensions）

### `/tools`
- **説明**: AIが使用できるツール（ファイル読み書き、検索、コマンド実行など）の一覧を表示します。`/tools desc` で技術的な詳細が確認できます。

### `/skills`
- **説明**: エージェントの専門スキルを管理します（有効化、無効化、一覧表示）。
- **使い方**: `/skills list` で使えるスキルを確認し、特定の技術（Azureや特定のフレームワーク等）に関するタスクを投げる前に確認します。

### `/mcp` / `/extensions` / `/agents`
- **説明**: MCP（Model Context Protocol）サーバー、CLI拡張機能、サブエージェントを管理するためのコマンドです。
- **応用的な使い方**: 外部のAPIやデータベース、独自の社内ツールなどをGemini CLIに連携させる高度な設定を行う際に使用します。

### `/commands`
- **説明**: ユーザー定義のカスタムコマンドを管理します。`.toml`ファイルで定義したコマンドを読み込み直す場合は `/commands reload` を使います。

## 🛠️ 6. ワークフローとユーティリティ（Workflow & Utilities）

### `/plan`
- **説明**: AIを「プランモード（設計モード）」に切り替えます。コードを書かずに、設計や調査だけを安全に行わせることができます。
- **現実的な使い方**: いきなりコードを書かせず、「`/plan 新しい認証機能の実装手順を考えて`」と指示して、方針をすり合わせるために使います。

### `/stats`
- **説明**: セッション中のAPIトークンの消費量や、ツールの使用回数などの統計情報を表示します。

### `/shells`
- **説明**: バックグラウンドで実行中のシェルプロセスの状態を表示します。
- **使い方**: サーバーの起動状態や、時間のかかるビルドコマンドの進捗を確認します。

### `/bug [見出し]`
- **説明**: Gemini CLI本体のバグをGitHub Issueとして報告します。

## 💡 7. 便利な構文（コマンド以外）

対話プロンプトで以下の構文も使えます：

### `@<ファイルパス>`
- **説明**: 指定したファイルやフォルダの中身を、プロンプトに直接挿入（読み込み）します。
- **現実的な使い方**: 「`@src/main.ts このコードのリファクタリングをして`」のように、見せたいファイルを明示的に指定して指示を出します。

### `!<コマンド>`
- **説明**: CLIを終了せずに、通常のシェルコマンド（ターミナルコマンド）を実行します。
- **現実的な使い方**: 「`!git status`」や「`!ls -la`」など、ちょっとした確認を行いたい時に非常に便利です。
